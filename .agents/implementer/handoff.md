# Handoff Report

## Observation
- We observed that `src/app/api/tools/text-to-image/route.ts`, `reimagine/route.ts`, `cleanup/route.ts`, `upscale/route.ts`, and `uncrop/route.ts` used the Clipdrop API.
- The user requested refactoring these to use the Hugging Face Free Inference API, specifying `stabilityai/stable-diffusion-xl-base-1.0` for text-to-image, `lambdalabs/sd-image-variations-diffusers` for reimagine, `runwayml/stable-diffusion-inpainting` for cleanup and uncrop, and `stabilityai/stable-diffusion-x4-upscaler` for upscale.
- Uncrop required outpainting, so `sharp` was used to pad the image with white background and generate a binary mask where the padded area is white (to be inpainted) and the original area is black (to be preserved).

## Logic Chain
- Replaced the endpoint URLs with `https://api-inference.huggingface.co/models/<MODEL_ID>`.
- Replaced `x-api-key` with `Authorization: Bearer ${process.env.HUGGINGFACE_API_KEY}`.
- For `text-to-image`, structured the payload as JSON with `{ inputs: prompt }`.
- For `reimagine`, configured raw binary upload (the buffer) directly to the API endpoint with `Content-Type` matching the image file type.
- For `cleanup`, formatted the image and mask as base64 and constructed a JSON payload with `{ inputs: 'background', image: base64_img, mask_image: base64_mask }`.
- For `upscale`, passed base64 encoded image and a JSON payload with `inputs: 'high resolution, high quality'`.
- For `uncrop`, logic was implemented using `sharp` to extend the image canvas and create a compatible binary mask, passed via JSON base64 formatting as required by `stable-diffusion-inpainting`.
- Webpack errors occurred for Next.js build relating to `sharp`, so `sharp` was added to `serverComponentsExternalPackages` in `next.config.js`.

## Caveats
- Hugging Face Free API can occasionally return 503 while models load, or 401 if the token isn't properly authorized for a specific model. The implementation matches standard Diffusers/Hugging Face Inference payloads. 
- The error generated by `npm run build` early on was due to Webpack's client-side bundling of node environment codes like `onnxruntime-node`. It seems tied to the `sharp` and `@imgly/background-removal` installation. Adding `serverComponentsExternalPackages` helps prevent server code from leaking.

## Conclusion
- The APIs have been fully refactored to use Hugging Face Free Inference API per specifications.
